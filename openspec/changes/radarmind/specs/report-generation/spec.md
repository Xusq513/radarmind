## 新增需求

### 需求：Markdown 日报生成
系统 SHALL 生成 Markdown 格式的日报，包含过滤后的文章，含标题、描述、链接和分数。

#### 场景：文章报告生成
- **WHEN** 评分后有过滤文章
- **THEN** 系统调用 LLM，传入生成提示词和所有文章数据，生成 markdown 报告

#### 场景：无文章报告生成
- **WHEN** 过滤结果为空文章
- **THEN** 系统生成 "今日无相关更新" 的 markdown 格式消息

### 需求：可配置生成提示词
系统 SHALL 使用可配置的提示词模板进行 LLM 日报生成。

#### 场景：自定义生成提示词
- **WHEN** 配置有 `"generation_prompt": "你是行业简报助手..."`
- **THEN** 系统使用该提示词模板，配合 Jinja2 变量替换

#### 场景：默认生成提示词
- **WHEN** 配置无 generation_prompt
- **THEN** 系统使用默认提示词，请求生成简洁的 markdown 摘要

### 需求：文件输出
系统 SHALL 将生成的报告写入 `output/YYYY-MM-DD.md`，文件名使用当前日期。

#### 场景：成功写入文件
- **WHEN** 报告成功生成
- **THEN** 系统将内容写入 `output/2026-05-26.md` 并记录成功消息

#### 场景：创建输出目录
- **WHEN** 输出目录不存在
- **THEN** 系统在写入文件前创建该目录
