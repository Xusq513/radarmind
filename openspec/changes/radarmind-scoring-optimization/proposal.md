## 为什么需要这个变更

当前 AI 评分采用串行方式：每篇文章对每个分类单独调用 LLM。30篇文章 × 6个分类 = 180次 API 调用，耗时几分钟。严重影响用户体验。

## 做了什么变更

- **优化评分调用方式**：从单文章单分类串行调用改为批量评分，所有文章和分类一次性发给 LLM，调用次数从 180 次降至 1 次
- **简化评分输入**：只用新闻标题进行评分，大幅减少 token 消耗
- **增强错误处理**：失败重试 3 次，超过则抛出异常
- **修改 ai-scoring 能力**：评分流程和接口发生变化

## 能力范围

### 新增能力

- **batch-scoring**: 批量评分能力，支持一次性对多篇文章多分类评分

### 修改的能力

- **ai-scoring**: 评分逻辑从串行单分类改为批量多分类，prompt 格式变化，输入只用标题

## 影响范围

- `src/radar_mind/scorer.py` 需要重写评分逻辑
- `prompts/scoring.j2` 需要更新 prompt 模板
- 评分延迟从分钟级降到秒级
- 重试机制变更（单次调用失败重试 3 次）
