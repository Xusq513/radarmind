## 修改的需求

### 需求：多分类评分（变更）

**变更说明**：从串行单分类评分改为批量多分类评分。

旧行为（串行）：
- 对每篇文章的每个分类单独调用 LLM
- 返回 {分类: 分数} 的字典

新行为（批量）：
- 对所有文章和分类在单次 LLM 调用中完成评分
- 返回包含多个元素的数组，每个元素对应一篇文章的分数

#### 场景：批量评分返回数组
- **WHEN** 有多篇文章需要评分
- **THEN** LLM 返回 JSON 数组，每个元素包含所有分类的分数

#### 场景：分数索引对应
- **WHEN** LLM 返回的数组为 [score1, score2, ...]
- **THEN** score1 对应第 1 篇文章，score2 对应第 2 篇文章，以此类推

### 需求：分数解析（变更）

**变更说明**：分数来自批量返回的 JSON 数组，不再是单个数字字符串。

旧行为：
- LLM 返回 "8" 或 "Score: 8" 格式
- 系统提取数字并验证

新行为：
- LLM 返回 JSON 数组，如 [{"AI": 8, "自动驾驶": 3}, ...]
- 系统解析 JSON 并对每个分数进行范围验证（1-10）

#### 场景：批量结果解析
- **WHEN** LLM 返回批量评分结果
- **THEN** 系统解析 JSON 数组并提取每个分类的分数

#### 场景：无效分数处理
- **WHEN** 批量结果中某个分数无效（非数字或超出范围）
- **THEN** 该分类分数设为 0
