## 新增需求

### 需求：批量评分
系统 SHALL 在单次 LLM 调用中对所有文章和所有分类进行评分，返回分数列表。

#### 场景：批量评分成功
- **WHEN** 有 30 篇文章和 6 个分类需要评分
- **THEN** 系统在单次 LLM 调用中完成所有评分，返回包含 30 个元素的数组

#### 场景：批量评分返回格式
- **WHEN** LLM 返回 JSON 数组
- **THEN** 系统解析数组，每个元素为 {分类: 分数} 的字典

#### 场景：分数范围验证
- **WHEN** LLM 返回的分数超出 1-10 范围
- **THEN** 系统将分数 clamping 到 1-10 范围内

### 需求：重试机制
系统 SHALL 在评分失败时最多重试 3 次，超过则抛出异常。

#### 场景：JSON 格式错误重试
- **WHEN** LLM 返回的内容无法解析为 JSON
- **THEN** 系统重试，最多 3 次

#### 场景：结构错误重试
- **WHEN** LLM 返回的 JSON 结构不符合预期（如缺少分类字段）
- **THEN** 系统重试，最多 3 次

#### 场景：超过重试次数
- **WHEN** 重试 3 次后仍然失败
- **THEN** 系统抛出 ScoringError 异常

#### 场景：认证错误不重试
- **WHEN** LLM API 返回 401 或 403 错误
- **THEN** 系统立即抛出异常，不进行重试

### 需求：标题输入
系统 SHALL 只使用新闻标题进行评分，不包含描述或链接。

#### 场景：标题评分
- **WHEN** 系统对文章进行评分
- **THEN** 输入给 LLM 的内容只包含标题，不包含 description 或 link
